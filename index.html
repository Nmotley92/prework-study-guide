HTML

  • The head element contains information, or metadata, that helps the browser to render the page correctly.
  • The body element represents the visible content shown to the user.
  • The tab athe the top of the browser displays the title.
  • Hypertext Markup Language (HTML) is one of the three cornerstone languages used in web development
  • HTML is the framework to the page. It is crucial to making the page user friendly.
  • The style element defines the CSS styles associated with a website.
  • The element defines metadata such as the character set, description, keywords, author, and viewport.
  • There should be only one main header per page.
  • Order matters

CSS

  • The most common way to style a webpage is by using an external CSS style sheet.
  • Separation of concerns is based on the idea that each section of code should have its own responsibility.
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property
  • Padding property adds space around the content inside an element.
  • Margin property indicates how much space we want around the outside of an element.
  • CSS box model
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • MVP stands for minimum viable product. This is usually the first iteration of a product without extra features.
  • Version control is the practice of tracking changes to our code.
  • Git is a version control system that lets you manage and keep track of each version of your project.
  • GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories that are stored remotely.
  • Using GitHub not only allows us to store our code, but also to easily collaborate with others.
  • Issues track, organize, and communicate ideas, tasks, bugs, and enhancements.
  • Identify bottlenecks or blockers that could prevent work progress.
  • Avoid duplicating work by assigning tasks or issues.
  • The default branch is the main branch.
  • The feature branch is where changes are being made.
  • In order to comepletley merge branches a pull request must first be submitted.
  • Pull request will almost always be reviewed by team meambers.
  • If the pull request is only once the pull request is reviewed and ok'd can the merge be finished.
